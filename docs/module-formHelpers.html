(inner) ValidationError(children, ariaName, classNameopt, messageopt, isUserErroropt)

It should always be the only item in its row - block content. Currently meant to be used with an input row to give short instructions or user error message. It also attempts to conform to accessibility:

Parameters:
Name Type Attributes Description
children string | object

Objects react can render.
ariaName string

ID of node this message will be describing.
className string <optional>

Optional classes to add to allow access with css.
message string | object | null <optional>

To be rendered above the children.
isUserError boolean <optional>

Is this an error message? Another option would be type instead - descriptor(s) for the message that will currently be turned into classes. Things like 'invalid'.
